<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Route `:id` -> saved-city lookup | Untrusted, user-controllable deep-link param; only used to LOOK UP a saved city, never to build a request |
| Client -> Open-Meteo forecast host | New read-only hourly GET (`hourly=temperature_2m,precipitation`) via the existing shared axios `http` client |
| Open-Meteo response -> HourlyForecast | Third-party JSON normalized into a locally-typed shape |
| i18n messages -> chart labels | Author-controlled static strings rendered to the chart canvas |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Tampering | Route `:id` feeding `useHourlyForecast` | mitigate | `:id` only resolves an existing saved city (existing `city` computed); the hourly fetch uses that city's stored lat/lon, never the raw param. Pass the resolved `city` computed - do not thread the param into the request |
| T-06-04 | Tampering (injection) | `fetchHourlyForecast` query params | mitigate | All query values passed via the axios `params` object (URL-encoded), never string-concatenated (ASVS V5) |
| T-06-05 | Denial of Service / robustness | Malformed Open-Meteo hourly response | mitigate | Locally-typed `HourlyForecastResponse` (no `any`); a missing/failed response surfaces as an empty/errored Vue Query state (`v-if="hourly"` hides the chart), not a crash |
| T-06-06 | Tampering (XSS) | Hourly chart time/temp/precip labels | mitigate | Chart.js renders labels to a canvas (not the DOM); static i18n messages interpolated as text; no `v-html` |
| T-06-SC | Tampering (supply chain) | npm/pip/cargo installs | accept | Phase 6 installs ZERO dependencies (RESEARCH Package Legitimacy Audit: N/A); no install task in this plan |
</threat_model>
