<objective>
Close the last English-only strings: move the three CitySearch yup validation messages into
i18n-keyed, locale-reactive form validation (I18N-04), and move the 28 WMO condition labels (+
fallback) out of `src/lib/wmo.ts` into `wmo.*` message keys rendered at the card and forecast-list
sites (I18N-05). No new dependency: `useField`'s `rules` is `MaybeRef`, so a `computed` yup schema
referencing `t()` re-validates on locale change; the WMO icon map stays in the util while the human
label becomes a message key.

Purpose: every user-facing string now lives in the locale files, so a language switch fully
localizes the app - and a small parity test proves en/ja stay in sync (the `wmo` block alone adds
29 keys per locale).
Output: a computed-schema CitySearch, a `labelKey`-based `wmo.ts` with updated render sites,
`validation.*` + `wmo.*` blocks in en.ts/ja.ts, and an en/ja key-parity test.
</objective>
