<objective>
Build the data backbone for the first weather slice: register Pinia and TanStack Vue
Query in the app, define shared weather types, create the Open-Meteo axios client
(geocoding + current weather), a WMO-code lookup util, the in-memory saved-cities Pinia
store (add/remove/dedupe), and the `useCurrentWeather` Vue Query composable that cards
will consume in plan 02-02.

Purpose: Each library gets one obvious job — axios = HTTP, Vue Query = server-state
caching/loading/error, Pinia = saved cities. This plan defines the contracts the UI
slice (02-02) depends on, so the executor of 02-02 receives types and function
signatures rather than exploring the codebase.

Output: package deps installed; `src/types/weather.ts`, `src/lib/openMeteo.ts`,
`src/lib/wmo.ts`, `src/stores/cities.ts`, `src/composables/useCurrentWeather.ts`, a
store unit test, and updated `src/main.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → geocoding query | Free-text city name flows into an outbound HTTP request URL param |
| Open-Meteo response → app state | Untrusted external JSON enters typed app objects and (in 02-02) the DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `geocodeCity` query param | mitigate | Pass the city name via axios `params` (axios URL-encodes), never string-concatenate into the URL; length/non-empty bound is enforced by vee-validate+yup in plan 02-02 before the call. |
| T-02-02 | Information Disclosure | error surfaces | accept | Public no-key API; no secrets/tokens exist to leak. Errors are mapped to generic UI text in 02-02; raw error objects are not rendered. |
| T-02-03 | Denial of Service | debounced/duplicate requests | mitigate | `geocodeCity`/`fetchCurrentWeather` accept an `AbortSignal`; Vue Query passes its signal so stale in-flight requests cancel. Debounce of the search is added in 02-02. |
| T-02-SC | Tampering | npm installs (pinia, axios, @tanstack/vue-query, vee-validate, yup) | mitigate | All five are on the pre-approved PROJECT.md list and are mainstream, widely-used packages; no `[ASSUMED]`/`[SUS]`/`[SLOP]` packages introduced. No blocking checkpoint required. |

Block-on: high severity only. No high-severity threats in this frontend-only, no-backend, no-key slice.
</threat_model>
