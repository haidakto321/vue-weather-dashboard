<objective>
Make the existing daily forecast chart honest across theme and language: its axis text, grid,
and legend follow the active Vuetify theme (CHRT-03), and its dataset labels + axis dates are
i18n-keyed and locale-formatted (CHRT-04). This is the flagship fix - the current
`ForecastChart.chartOptions` is a plain non-reactive object, so the chart never restyles on a
theme flip. No new chart, no new data: one component edit plus two i18n keys.

Purpose: charts must read the same live theme/locale sources the rest of the app already uses,
so a study reader sees Vuetify's theme and vue-i18n's locale each doing one visible job on a
real chart.
Output: a theme-and-locale-reactive `ForecastChart.vue`, plus a new `chart` message block
(`tempHigh`, `tempLow`) in en.ts and ja.ts.
</objective>

| Boundary | Description |
|----------|-------------|
| i18n message store -> chart labels | Author-controlled static TS strings displayed on the chart |
| forecast prop -> chart datasets | Already-typed `DailyForecast` arrays (validated upstream by `fetchForecast`) mapped to Chart.js data |

Note: this plan adds NO new network call and NO new user input. It only reads existing reactive
sources (Vuetify theme, vue-i18n locale/unit) and displays strings. Surface is minimal.
