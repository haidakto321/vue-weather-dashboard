<interfaces>
VueUse useGeolocation (installed @vueuse/core 14.3.0, verified against node_modules/@vueuse/core/dist/index.js line 4329): useGeolocation(options) returns { isSupported, coords, locatedAt, error, resume, pause }. With immediate:false, nothing runs until resume() is called. resume() calls navigator.geolocation.watchPosition(...) - a CONTINUOUS watch, not a one-shot getCurrentPosition - so pause() must be called on the first fix to stop it (Pitfall 1). coords.value starts as { latitude: Infinity, longitude: Infinity, ... } before any fix - guard with Number.isFinite(coords.value.latitude), never a truthiness check (Pitfall 2, Infinity is truthy). error.value.code follows MDN's GeolocationPositionError: 1=PERMISSION_DENIED, 2=POSITION_UNAVAILABLE, 3=TIMEOUT.

src/stores/cities.ts addCity(geo: GeoCity) (lines 48-60): dedupes by cityKey() (id ? String(id) : `${lat},${lon},${name}` composite). Passing id: 0 makes cityKey() fall back to the lat,lon,name composite - already globally unique per geolocation fix, no store change needed for GEO-01 dedupe.

## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Browser Geolocation API -> app | precise user coordinates enter the app on explicit user action (button click) |
| Coordinates -> Open-Meteo | coordinates are later sent only to api.open-meteo.com, the same host every other saved city already uses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-GEO | Information Disclosure | Geolocation coordinates captured by useMyLocation | accept | Coordinates are captured only on explicit user click (immediate:false, no auto-locate on page load), stored only in the existing weather-cities localStorage key (no new destination, no analytics/telemetry added), and later sent only to api.open-meteo.com - the same pattern every other saved city already follows. |
| T-07-WATCH | Denial of Service (local) | useGeolocation's resume() starts a continuous watchPosition, not a one-shot fetch | mitigate | pause() is called immediately on the first finite coordinate fix or on any error (Pitfall 1), so the browser's location indicator does not stay active indefinitely; tryOnScopeDispose inside useGeolocation covers component unmount as a second layer. |
| T-07-XSS | Tampering | geo.myLocation is rendered via t() interpolation, never v-html | accept | Vue template interpolation auto-escapes; the string is a static, developer-authored i18n value, not user input. |
</threat_model>
